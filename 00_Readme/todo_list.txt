#### Data Quality — IAH Terminal D Split-Shift Hours Not Representable in Schema
- 2026-07-14: IAH's own page shows Terminal D running a split shift 4 of 7 days
  (e.g. 7:00a-11:30a + 4:30p-8:00p Sun/Mon/Wed/Fri; continuous 7:00a-8:00p
  Tue/Thu/Sat) — see the "IAH Terminal D TSA checkpoint hours" section of

  but this doesn't and can't fix the midday gap — the live `isOpen`/`isDisplayable`
  flags from the new JSON API are what actually protect against that now. Revisit
  if the schema ever needs day-of-week-aware hours (not just IAH D).

#### Task Scheduler — tsa_app_las_hours_monitor Needs LogonType Switched to Password
- 2026-07-22: Registered via PowerShell with `LogonType Interactive` because the tool shell
  used to create it wasn't admin-elevated (`Register-ScheduledTask` with `RunLevel Highest`
  or a `Password` logon type both returned "Access is denied" non-interactively). This means
  it will get torn down ~30s after any non-logon-event manual start (see
  project_windows_task_scheduler_gotchas item 3) and won't survive being started outside a
  real interactive logon. Open Task Scheduler GUI → this task's Properties → General tab →
  select "Run whether user is logged on or not" and enter the account password (matches
  tsa_app_scraper/tsa_app_bos_hours_monitor's existing Password logon type) — can't be done
  non-interactively since it requires the Windows password, which should never be requested
  in chat.
