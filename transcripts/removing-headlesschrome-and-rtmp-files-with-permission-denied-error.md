A few things to clarify before touching this:

What's happening: file.remove() doesn't accept a recursive argument — that parameter is silently ignored. So for directories (which HeadlessChrome* folders are), it fails with "Permission denied" because you can't remove a directory with file.remove() — you need unlink() with recursive = TRUE.

One assumption to confirm: Are any of these files/folders currently locked by a running Chrome process? If scraping is actively running, some will still be permission-denied regardless. This script should only be run when the scraper is idle.

Here's the corrected script:

r
# Delete HeadlessChrome and Rtmp temp files/folders ----
# Run only when scraper is NOT actively running.

temp_path <- "C:/Users/james/AppData/Local/Temp"

targets <- list.files(
  path       = temp_path,
  pattern    = "^(HeadlessChrome|Rtmp)",
  full.names = TRUE
)

if (length(targets) == 0) {
  message("No matching files or folders found.")
} else {
  results <- unlink(targets, recursive = TRUE, force = TRUE)
  
  n_success <- sum(results == 0)
  n_fail    <- sum(results != 0)
  
  message(glue::glue("{n_success} item(s) deleted, {n_fail} item(s) failed (likely locked by active process)."))
}
Key changes:

unlink() instead of file.remove() — handles both files and directories
recursive = TRUE — actually works here (deletes folder contents first)
force = TRUE — bypasses read-only flags where possible
Result checking — unlink() returns 0 on success, non-zero on failure, so you can see what actually got removed vs. what's still locked
